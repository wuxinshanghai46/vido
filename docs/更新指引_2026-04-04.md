# VIDO AI 平台更新指引 — 2026-04-04

## 一、架构改进

### 1. 数据库模块化存储
将原来的单一 `vido_db.json` 拆分为按功能模块独立的 JSON 文件：

| 文件 | 模块 | 数据 |
|------|------|------|
| `project_db.json` | 视频项目 | 项目 + 剧情 + 片段 + 成片 |
| `avatar_db.json` | 数字人 | 数字人视频任务 |
| `comic_db.json` | 漫画 | 漫画生成任务 |
| `novel_db.json` | 小说 | AI 小说数据 |
| `portrait_db.json` | 形象 | 角色形象生成 |
| `i2v_db.json` | 图生视频 | 图生视频任务 |
| `asset_db.json` | 素材库 | 用户上传素材 |
| `publish_db.json` | 发布 | 社媒发布记录 |
| `voice_db.json` | 声音 | 自定义声音克隆 |

- 首次启动自动从旧 `vido_db.json` 迁移，旧文件备份为 `vido_db_backup.json`
- API 接口完全兼容，无需修改前端调用

## 三、TTS 语音系统重构

### 回退链修复
- **问题**：原来任何一个 TTS 供应商失败就直接返回 null，后续供应商永远不会被尝试
- **修复**：改为逐个 try-catch 回退链，每个供应商失败后自动尝试下一个
- **日志**：每次 TTS 调用会打印使用了哪个供应商

### 自定义声音克隆
- **上传**：`POST /api/workbench/upload-voice` 上传音频（10-30 秒清晰录音）
- **存储**：声音元数据存入 `voice_db.json`，文件存入 `outputs/voices/`
- **克隆**：
  - 优先用 Fish Audio API（需配置 key）→ 上传参考音频 → 获取 reference_id → 后续直接使用
  - 回退：基础 TTS + FFmpeg 音色微调
- **列表**：自定义声音出现在声音选择器的 "🎤 我的声音" 分组

### 可用声音汇总
共 22+ 个声音：智谱7 + 讯飞7 + OpenAI6 + Windows2 + 用户自定义

## 四、数字人模块增强

### 网络/限流重试
- TLS 断连 → 自动重试 5 次（间隔 10-50 秒）
- 429 限流（访问量过大）→ 自动重试 5 次（间隔 30-75 秒）
- 前端实时显示重试状态

### 分段优化
- 每段目标：30-50 字（约 8-12 秒说话），匹配 CogVideoX 10 秒基础视频
- 上限从 100 字降至 60 字，避免视频循环感
- 超过 50 字自动触发 AI 分段

### 历史记录持久化
- 数字人历史从数据库加载（原来仅内存，刷新丢失）
- 任务列表合并内存 + 数据库，重启后不丢失
- 保存 ratio/model 信息供历史展示

## 五、运镜/镜头控制系统

### 19 种运镜类型
| 类别 | 运镜 |
|------|------|
| 基础 | 固定、推、拉、左摇、右摇、上仰、下俯 |
| 进阶 | 跟拍、希区柯克变焦、环绕360°、升、降、手持 |
| 电影级 | 第一人称、过肩、航拍、甩镜头、缓推、子弹时间 |

### 11 种景别
大远景、远景、全景、中景、中近景、特写、大特写、仰拍、俯拍、鸟瞰、荷兰角

### 使用方式
1. 在场景属性面板选择"镜头运动"和"景别"
2. 自动注入对应的英文 prompt 到视频生成
3. AI 生成的 camera 字段作为回退

## 十、多格式导出

下载视频支持格式参数：`?format=mp4|webm|gif|mov`

| 格式 | 编码 | 用途 |
|------|------|------|
| MP4 | H.264+AAC | 默认，通用 |
| WebM | VP9+Opus | 网页嵌入 |
| GIF | 12fps 480p | 社交媒体预览 |
| MOV | H.264+AAC | Apple 生态 |

- 首次转换由 FFmpeg 实时处理，结果缓存到 `outputs/exports/`

---

## 十一、视频预览/下载一致性修复

- `getFinalVideoByProject()` 改为返回最新记录（原来返回最旧的）
- 流式播放添加 `Cache-Control: no-cache` + `ETag` 防止浏览器缓存
- FFmpeg 合并时添加 `-map 0:a?` 保留可选音轨
